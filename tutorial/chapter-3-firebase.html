Chapter 3

Firebase configuration & Realtime Data

A sports analytics app is useless without data. We will use Google Firebase to provide a realtime NoSQL database (Firestore) and authentication for our Admin Panel. This chapter walks you through connecting your frontend to a Firebase backend.

1. Setting up the Firebase Project

  1. Go to the Firebase Console.
  2. Click Add project and name it something like Phase4FC-App.
  3. Disable Google Analytics for now.
  4. Once created, click the Web </> icon to register your app. Keep the default settings and click Register App.

Firebase will provide you with a configuration object that looks like this: const firebaseConfig = { apiKey: "...", ... }. Copy this code block!

2. Initializing Firebase Configuration

In our `js` folder, we created a file called firebase-config.js. This acts as our bridge to the backend database. Paste your config object into this file along with the necessary module imports.

Open js/firebase-config.js and add:

// Import the core Firebase App module
import { initializeApp } from "https://www.gstatic.com/firebasejs/11.1.0/firebase-app.js";

// Import Optional Firebase services (Firestore for data, Auth for login)
import { 
    getFirestore, collection, addDoc, updateDoc, deleteDoc, doc, onSnapshot, query, orderBy
} from "https://www.gstatic.com/firebasejs/11.1.0/firebase-firestore.js";

import { 
    getAuth, signInWithEmailAndPassword, onAuthStateChanged, signOut
} from "https://www.gstatic.com/firebasejs/11.1.0/firebase-auth.js";

// !!! REPLACE THIS WITH YOUR CONFIG OBJECT FROM STEP 1 !!!
const firebaseConfig = {
    apiKey: "YOUR_API_KEY",
    authDomain: "phase4fc-app.firebaseapp.com",
    projectId: "phase4fc-app",
    storageBucket: "phase4fc-app.firebasestorage.app",
    messagingSenderId: "1234567890",
    appId: "1:1234567890:web:abcdef123456"
};

// Initialize Firebase
const app = initializeApp(firebaseConfig);

// Initialize Cloud Firestore and get a reference to the service
const db = getFirestore(app);

// Initialize Firebase Authentication
const auth = getAuth(app);

// Export these functions and variables so we can use them in app.js
export { 
    app, db, auth, 
    collection, addDoc, updateDoc, deleteDoc, doc, onSnapshot, query, orderBy,
    signInWithEmailAndPassword, onAuthStateChanged, signOut
};

Why the CDN Links?

Since we aren't using a bundler like Webpack or Vite, we import Firebase modules directly via CDN links (e.g. https://www.gstatic.com/...). This keeps our project simple and runnable directly in the browser.

3. Setting up Firestore Database Rules

Before our app can read data, we need to create the database.

  1. In the Firebase Console menu (left sidebar), click Firestore Database under Build.
  2. Click Create database. Start in Test mode to allow our app to read and write without logging in right away. Click Next, choose a location, and click Enable.

Security Warning

Test mode leaves your database completely open to the public for 30 days. In Chapter 5, we will implement Firebase Auth to lock down write permissions.

Data Collections Needed

For Phase 4 FC, we will use two primary collections (think of these as tables in a SQL database):

  • players: Stores player names, positions, and aggregate stats.
  • sessions: Stores individual match/training session data including scores, locations, and individual player performance for that specific day.

You can manually add a dummy player directly in the Firebase console for now simply to test that it works in the next chapter!
