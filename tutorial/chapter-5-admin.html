Chapter 5

Admin Panel & Authentication

To securely manage our club's stats, we need an Admin Panel that is protected by a login screen. We will use Firebase Authentication (Email/Password) to ensure only authorized coaches can add matches or edit player details.

1. Enable Firebase Auth

  1. Go back to your Firebase Console.
  2. Click Authentication > Get Started.
  3. Select the Email/Password provider and enable it.
  4. Go to the Users tab and manually add a user (e.g., coach@phase4fc.com with a strong password).

2. Building Admin UI and Login Modal

Create `admin.html`. We will wrap our entire admin dashboard inside a div (`#admin-content`) that is hidden by default. We will also create a Login modal (`#admin-login-modal`) that is visible by default.

3. Javascript Auth Logic

We need to add authentication handling to `js/app.js`. Firebase provides a helper called onAuthStateChanged which acts like a listener, instantly telling us if a user is logged in or out across page reloads. 

// In js/app.js
import { auth, signInWithEmailAndPassword, onAuthStateChanged, signOut } from './firebase-config.js';

// Add this to your State Object
const State = {
    // ... other properties
    user: null, // Keep track of auth state
    
    initAuth() {
        // This listener fires on load AND whenever login/logout happens
        onAuthStateChanged(auth, (user) => {
            this.user = user;
            
            // Grab UI elements
            const adminContent = document.getElementById('admin-content');
            const loginModal = document.getElementById('admin-login-modal');

            // Only run UI toggles if we are currently on the admin page
            if (window.location.pathname.includes('admin.html')) {
                if (user) {
                    // Logged IN
                    if (loginModal) loginModal.classList.add('hidden');
                    if (adminContent) adminContent.classList.remove('hidden');
                } else {
                    // Logged OUT
                    if (loginModal) loginModal.classList.remove('hidden');
                    if (adminContent) adminContent.classList.add('hidden');
                }
            }
        });
    }
};

// Global Login Function called by our HTML button
window.handleLogin = async function() {
    const email = document.getElementById('login-email').value;
    const pass = document.getElementById('login-password').value;
    const errorMsg = document.getElementById('login-error');
    
    try {
        errorMsg.classList.add('hidden');
        // Firebase handles the actual secure check!
        await signInWithEmailAndPassword(auth, email, pass);
        // onAuthStateChanged will automatically fire and show the dashboard!
    } catch (error) {
        errorMsg.textContent = "Invalid Credentials. Try again.";
        errorMsg.classList.remove('hidden');
    }
}

// Global Logout Function
window.handleLogout = async function() {
    await signOut(auth);
}

// Don't forget to call initAuth inside State.init()!

Security Note

Hiding elements with CSS (.hidden) does not actually secure your database. A smart user could remove the hidden class using browser dev tools! True security comes from returning to the Firebase Console and changing your Firestore Rules to only allow writes if request.auth != null.
