Chapter 1

Project Setup & Configuration

Welcome to the first chapter! We will start by creating the basic folder structure of our application and setting up Tailwind CSS, the utility-first CSS framework that will power our design system.

1. Folder Structure

Create a new folder on your computer named phase4fc. Open this folder in your favorite code editor (like VS Code). Inside, create the following empty files and folders:

phase4fc/
├── css/
│   └── styles.css
├── js/
│   ├── analytics.js
│   ├── app.js
│   └── firebase-config.js
├── index.html
├── matches.html
├── admin.html
└── stats/
    ├── stats.html
    └── player-comparison.html

2. The Core Setup (index.html)

Let's initialize our index.html file. Since this project uses Tailwind CSS via CDN, we don't need to run complex build processes. We simply include the Tailwind script tag and define our custom brand colors using `tailwind.config`.

Open index.html and add the following code:

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Phase 4 FC | Club Analytics</title>

    <!-- 1. Load Tailwind CSS via CDN -->
    <script src="https://cdn.tailwindcss.com"></script>
    
    <!-- 2. Load our custom stylesheet (we will edit this later!) -->
    <link rel="stylesheet" href="css/styles.css">

    <!-- 3. Configure Custom Tailwind Brand Colors -->
    <script>
        tailwind.config = {
            theme: {
                extend: {
                    colors: {
                        primary: '#10b981',   /* Emerald Green */
                        secondary: '#0ea5e9', /* Sky Blue */
                        accent: '#f59e0b',    /* Amber/Gold */
                        dark: '#0f172a',      /* Deep Navy blue background */
                        card: '#1e293b',      /* Slightly lighter blue for cards */
                    }
                }
            }
        }
    </script>
</head>

<!-- 4. Apply the dark background and light text to the entire body -->
<body class="bg-dark text-slate-50">

    <div class="p-8 text-center">
        <h1 class="text-4xl font-bold text-primary">Hello Phase 4 FC!</h1>
        <p class="text-slate-400 mt-4">Tailwind is successfully configured.</p>
    </div>

</body>
</html>

Why the Config Script?

By customizing the Tailwind configuration directly in the HTML file, we generate utility classes like text-primary or bg-dark on the fly. This gives us complete control over our design system's color palette without writing custom CSS.

3. Initializing Global Styles

While Tailwind handles 95% of our styling, we still want a global stylesheet to define foundational elements like fonts and custom glassmorphism effects.

Open css/styles.css and add:

/* Import Google Font (Inter) for a modern, clean typography */
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700;900&display=swap');

/* Apply font globally */
body {
    font-family: 'Inter', sans-serif;
}

/* 
 * Custom Glassmorphism Utility Class 
 * This creates a semi-transparent, blurred background effect 
 * that we will use heavily for our cards and navigation bar.
 */
.glass-card {
    background: rgba(30, 41, 59, 0.4); /* Matches text-card with opacity */
    backdrop-filter: blur(12px);
    -webkit-backdrop-filter: blur(12px);
    border: 1px solid rgba(255, 255, 255, 0.05);
}

/* Smooth scrolling behavior for entire page */
html {
    scroll-behavior: smooth;
}

Test Your Setup

Open index.html in your browser. You should see a dark blue screen with green text saying "Hello Phase 4 FC!". If you do, setup is complete and working correctly!
