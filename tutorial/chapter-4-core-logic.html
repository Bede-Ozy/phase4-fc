Chapter 4

Core App Logic & State

Most modern single-page applications heavily rely on a State Management System. Instead of directly manipulating HTML whenever data changes, we update our "State" variable. The app automatically detects this change and re-renders the necessary HTML chunks.

1. Setting up app.js

Open js/app.js. First, we need to import our Firebase configuration so we can read from the database.

import {
    db, collection, doc, onSnapshot, query, orderBy
} from './firebase-config.js';

// We also import a companion file called analytics.js (which we'll create later for calculating stats points)
// import Analytics from './analytics.js';

2. The State Object

Next, we create a global State object. It holds our data arrays (players and sessions), any active filters, and our core initialization methods.

const State = {
    // 1. Core Data
    players: [],
    sessions: [],
    
    // 2. Initializer Method
    init() {
        this.subscribeToData(); // Listen for database changes
    },

    // 3. Realtime Database Subscription
    subscribeToData() {
        // Query players collection alphabetically
        const qPlayers = query(collection(db, "players"), orderBy("name"));
        
        // onSnapshot fires immediately AND every time data changes in the DB
        onSnapshot(qPlayers, (snapshot) => {
            // Map the Firebase Document Snapshot back into a normal Javascript Array
            this.players = snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() }));
            
            // Re-render the UI with the new data!
            this.render();
        });

        const qSessions = query(collection(db, "sessions"));
        onSnapshot(qSessions, (snapshot) => {
            this.sessions = snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() }));
            this.render();
        });
    },

    // 4. Main Router/Render Function
    render() {
        this.renderLatestMatch();
        this.renderMatchList();
    },

    // 5. HTML Injection Methods
    renderMatchList() {
        const container = document.getElementById('match-list');
        if (!container || this.sessions.length === 0) return;
        
        // Sort sessions by date (newest first)
        const sorted = [...this.sessions].sort((a, b) => new Date(b.date) - new Date(a.date));
        
        // Map data to HTML strings and inject them into the DOM
        container.innerHTML = sorted.map(s => {
            const t1 = s.teams[0];
            const t2 = s.teams[1];
            
            return `
            <div class="glass-card p-3 rounded-2xl flex flex-col gap-3 mb-4 border border-slate-700 hover:border-primary/30 transition-all cursor-default">
                <div class="flex items-center gap-2">
                    <div class="text-[9px] font-black text-slate-500 uppercase tracking-widest">${new Date(s.date).toLocaleDateString()}</div>
                    <span class="w-1.5 h-[1px] bg-slate-700"></span>
                    <div class="text-[9px] font-black text-primary uppercase tracking-widest">${s.type}</div>
                </div>
                
                <div class="flex items-center justify-between bg-slate-800/40 p-3 rounded-xl border border-slate-700/30">
                    <div class="font-bold text-sm truncate">${t1.name}</div>
                    <div class="flex items-center gap-4 px-4 py-1.5 bg-slate-900 rounded-lg border border-slate-700 mx-2">
                        <span class="text-xl font-black text-primary">${t1.score}</span>
                        <span class="text-[10px] font-bold text-slate-600">vs</span>
                        <span class="text-xl font-black text-primary">${t2.score}</span>
                    </div>
                    <div class="font-bold text-sm truncate">${t2.name}</div>
                </div>
            </div>`;
        }).join(''); // Join array of HTML strings into one massive string
    },
    
    // (Stub for Latest Match)
    renderLatestMatch() {
        // Similar logic, grabbing sorted[0] and rendering its expanded stats
    }
};

// Start the Application!
State.init();

Understanding onSnapshot()

In a traditional REST API, you must hit a refresh button or write polling intervals to check if data changed on the server. Firebase's onSnapshot() opens a WebSocket connection. When your fellow admin saves a new match from their phone, onSnapshot() immediately triggers on your computer, updating this.sessions, and calling this.render(). The UI updates instantly across all connected screens globally.

3. Testing Dynamic Rendering

If you've properly set up Firebase and added a dummy session under the sessions collection manually in the Firebase console, refreshing your site should instantly replace the empty `<div id="match-list">` with a beautifully styled glass-card containing the match results!
